# GitHub Upload Checklist

Use this checklist before uploading or publishing the project.

## Include

- `football_analyst/`
- `tests/`
- `data/sample/`
- `outputs/statsbomb-england-mexico/`
- `README.md`
- `requirements.txt`
- `pyproject.toml`
- `.env.example`
- `.gitignore`
- `analyst.py`

## Do Not Include

- `.env`
- `.venv/`
- `venv/`
- `.pytest_cache/`
- `__pycache__/`
- `data/raw/`
- `data/processed/`
- any paid or licensed provider exports

## Suggested Repository Description

Football event-data knowledge graph using Neo4j, LangGraph and public StatsBomb open-data, with event-driven England vs Mexico tactical visualisations.

## Suggested LinkedIn Link Text

I built a small football knowledge graph to show how graph-based context can support AI reasoning beyond enterprise documents and workflows. The repo uses public historical StatsBomb data, so the analysis is event-pattern driven rather than a live scouting report.
